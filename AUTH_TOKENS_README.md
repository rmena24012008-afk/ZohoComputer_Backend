## Overview

The application uses a **two-layer token system**:

| Layer | Purpose | Technology | Lifetime |
|-------|---------|------------|----------|
| **JWT (Internal)** | Authenticates users to *this* API | HS256 signed JWT | 24 hours |
| **OAuth2 (External)** | Authenticates users to *third-party* providers (Google, GitHub, Slack, etc.) | OAuth2 access/refresh tokens | Provider-dependent (typically 1 hour + refresh) |

**Key security features:**
- Passwords are hashed with **BCrypt** (salt rounds auto-generated).
- OAuth2 `access_token`, `refresh_token`, and `client_secret` are encrypted at rest using **AES-256-GCM**.
- JWT secret and encryption key are loaded from environment variables / `secrets.properties` (never hardcoded).

### JWT Token Structure

Tokens are generated by `JwtService` using the **HS256** algorithm (HMAC-SHA256).

**Claims payload:**
```json
{
  "user_id": 1,
  "username": "johndoe",
  "email": "johndoe@example.com",
  "iat": 1705312200,
  "exp": 1705398600
}
```

| Claim | Type | Description |
|-------|------|-------------|
| `user_id` | long | User's primary key |
| `username` | string | User's display name |
| `email` | string | User's email address |
| `iat` | timestamp | Issued-at time (epoch seconds) |
| `exp` | timestamp | Expiration time (`iat` + 24 hours) |

**Signing key:** Derived from the `JWT_SECRET` environment variable using `Keys.hmacShaKeyFor()`. The secret must be at least 32 bytes (256 bits) for HS256.

### Token Validation & AuthFilter

The `AuthFilter` is a servlet filter mapped to `/api/*` that runs on every request.

**Behavior:**

1. **Skips** `OPTIONS` requests (CORS preflight — handled by `CorsFilter`).
2. **Skips** public paths: `/api/auth/login`, `/api/auth/register`.
3. **Extracts** the JWT from:
   - `Authorization: Bearer <token>` header (primary), OR
   - `?token=<token>` query parameter (fallback for browser file downloads).
4. **Validates** the token via `JwtService.validateToken()`.
5. **Sets** request attributes on success:
   - `userId` (`Long`) — used by all downstream servlets.
   - `username` (`String`) — available for logging/display.
6. **Returns `401`** if the token is missing, malformed, or expired.

**Filter chain order** (configured in `web.xml`):
```
CorsFilter  →  AuthFilter  →  Servlet
   /*              /api/*
```

## 2. OAuth2 Provider Token Management

These endpoints manage OAuth2 credentials for third-party providers (Google, GitHub, Slack, custom APIs, etc.). All require JWT authentication.

### POST `/api/auth/oauth/link` — Link Provider

Saves OAuth2 client credentials for a provider and optionally returns an authorization URL.

**Authentication:** ✅ Required

**Request Body:**
```json
{
  "provider": "google",
  "client_id": "123456789.apps.googleusercontent.com",
  "client_secret": "GOCSPX-abc123...",
  "token_endpoint": "https://oauth2.googleapis.com/token",
  "oauth_token_link": "https://accounts.google.com/o/oauth2/v2/auth",
  "scope": "https://www.googleapis.com/auth/calendar.readonly",
  "redirect_uri": "http://localhost:5173/oauth/callback",
  "header_type": "Bearer"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `provider` | string | ✅ | Provider identifier (e.g. `"google"`, `"github"`, `"slack"`) |
| `client_id` | string | ✅ | OAuth2 application client ID |
| `client_secret` | string | ✅ | OAuth2 application client secret (encrypted at rest) |
| `token_endpoint` | string | ✅ | Provider's token exchange URL |
| `oauth_token_link` | string | ❌ | Provider's authorization URL (needed for URL generation) |
| `scope` | string | ❌ | OAuth2 scopes to request |
| `redirect_uri` | string | ❌ | Redirect URI after authorization |
| `header_type` | string | ❌ | Authorization header scheme. Default: `"Bearer"`. Options: `"Bearer"`, `"Token"`, `"Basic"`, `"ApiKey"` |

**Success Response — `201 Created`:**
```json
{
  "success": true,
  "data": {
    "provider": "google",
    "status": "credentials_saved",
    "authorization_url": "https://accounts.google.com/o/oauth2/v2/auth?scope=...&client_id=...&state=google_1705312200000&response_type=code&redirect_uri=...&prompt=consent&access_type=offline"
  }
}
```

> **Note:** The `authorization_url` is only included if `oauth_token_link`, `scope`, and `redirect_uri` are all provided. The `state` parameter follows the format `<provider>_<timestamp>`.

**Upsert behavior:** If the `(user_id, provider)` pair already exists, the record is updated with the new credentials (`INSERT ... ON DUPLICATE KEY UPDATE`).

### OAuth2 Flow Walkthrough

Here's the complete end-to-end flow for linking a provider:

```
Step 1: Link Provider (save credentials + get auth URL)
─────────────────────────────────────────────────────────
Client  ──POST /api/auth/oauth/link──►  Backend
        { provider, client_id, client_secret,             
          token_endpoint, oauth_token_link,                
          scope, redirect_uri }                            
                                                           
Client  ◄── 201 { authorization_url: "https://..." } ──  Backend
                                                           

Step 2: User Authorization (browser redirect)
─────────────────────────────────────────────
Client  ──redirect──►  Provider Auth Page
                        (user logs in, grants consent)
Provider ──redirect──►  /api/auth/oauth/callback?code=...&state=...


Step 3: Code Exchange (automatic)
─────────────────────────────────
Backend ──POST token_endpoint──►  Provider Token API
        { grant_type=authorization_code,
          client_id, client_secret,
          code, redirect_uri }

Backend ◄── { access_token, refresh_token, expires_in } ── Provider
        (tokens encrypted → stored in DB)

Client  ◄── 200 { status: "tokens_exchanged" } ── Backend


Step 4: Use Tokens (auto-refresh)
─────────────────────────────────
Backend calls OAuthTokenService.getValidAccessToken(userId, provider)
  → If token expired AND refresh_token exists:
      → POST token_endpoint with grant_type=refresh_token
      → Update stored tokens in DB
  → Returns valid AuthToken with decrypted access_token
  → Builds auth header: "<header_type> <access_token>"
```

**Auto-refresh logic:**
- `OAuthTokenService.getValidAccessToken()` checks `expiresAt` against the current time.
- If expired and a `refresh_token` is available, it automatically refreshes the token.
- A 60-second safety buffer is applied to `expires_in` values (the token is treated as expired 60 seconds early).
- Some providers rotate refresh tokens on each refresh — the new refresh token is automatically persisted.

## 3. Token Encryption (At-Rest Security)

The `TokenEncryptionService` provides transparent AES-256-GCM encryption for sensitive fields.

### Encrypted Fields

| Column (in `auth_tokens`) | Encrypted? |
|---------------------------|------------|
| `access_token` | ✅ AES-256-GCM |
| `refresh_token` | ✅ AES-256-GCM |
| `client_secret` | ✅ AES-256-GCM |
| `client_id` | ❌ Plain text |
| `token_endpoint` | ❌ Plain text |
| `oauth_token_link` | ❌ Plain text |

### How It Works

```
Encryption:
  plaintext  →  SecureRandom(12-byte IV)  →  AES-256-GCM encrypt  →  Base64(IV || ciphertext || auth-tag)

Decryption:
  Base64 decode  →  split IV[0..12] + ciphertext[12..]  →  AES-256-GCM decrypt  →  plaintext
```

| Property | Value |
|----------|-------|
| Algorithm | AES/GCM/NoPadding |
| Key size | 256 bits |
| IV size | 96 bits (12 bytes), randomly generated per encryption |
| Auth tag | 128 bits (16 bytes) |
| Key derivation | SHA-256 hash of `ENCRYPTION_SECRET` env var |
| Storage format | `Base64( IV[12] ‖ CipherText ‖ AuthTag[16] )` |

**Key properties:**
- **Randomized IV:** Encrypting the same plaintext twice produces different ciphertext, preventing equality leakage.
- **Authenticated encryption:** GCM's auth tag detects any tampering with the ciphertext.
- **Null-safe:** `encrypt(null)` and `decrypt(null)` both return `null`.

### Transparency

Callers never handle encryption directly. The `AuthTokenDao` automatically:
- **Encrypts** `accessToken`, `refreshToken`, and `clientSecret` on `upsert()`.
- **Decrypts** the same fields on `findByUserAndProvider()` and `findByUser()`.
- The `AuthToken` model always holds **plain-text** values in memory.

## 4. Database Schema

### `auth_tokens` Table

```sql
CREATE TABLE IF NOT EXISTS auth_tokens (
    id               BIGINT        PRIMARY KEY AUTO_INCREMENT,
    user_id          BIGINT        NOT NULL,
    provider         VARCHAR(100)  NOT NULL,
    header_type      VARCHAR(50)   DEFAULT 'Bearer',
    access_token     TEXT          NOT NULL,          -- 🔒 AES-256-GCM encrypted
    refresh_token    TEXT          DEFAULT NULL,       -- 🔒 AES-256-GCM encrypted
    expires_at       TIMESTAMP     DEFAULT (NOW() + INTERVAL 1 HOUR),
    client_id        VARCHAR(255)  DEFAULT NULL,
    client_secret    TEXT          DEFAULT NULL,       -- 🔒 AES-256-GCM encrypted
    token_endpoint   VARCHAR(500)  DEFAULT NULL,
    oauth_token_link VARCHAR(1000) DEFAULT NULL,
    updated_at       TIMESTAMP     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    UNIQUE KEY uq_user_provider (user_id, provider)
);
```

**Indexes:**
```sql
CREATE INDEX idx_auth_tokens_user    ON auth_tokens(user_id);
CREATE INDEX idx_auth_tokens_expires ON auth_tokens(expires_at);
```

**Constraints:**
- `UNIQUE KEY uq_user_provider (user_id, provider)` — One token record per user per provider.
- `FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE` — Tokens are auto-deleted when the user is deleted.

### `users` Table (for reference)

```sql
CREATE TABLE IF NOT EXISTS users (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    username        VARCHAR(50) UNIQUE NOT NULL,
    email           VARCHAR(100) UNIQUE NOT NULL,
    password_hash   VARCHAR(255) NOT NULL,            -- BCrypt hashed
    preferences     JSON         DEFAULT NULL,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 5. Configuration & Environment Variables

All secrets are resolved via `AppConfig` with this priority:

```
Environment Variable  →  System Property  →  secrets.properties  →  Hardcoded Default
```

| Variable | Purpose | Required |
|----------|---------|----------|
| `JWT_SECRET` | HS256 signing key for JWT tokens. Must be ≥ 32 bytes. | ✅ |
| `ENCRYPTION_SECRET` | Key material for AES-256-GCM token encryption. SHA-256 hashed to derive the 256-bit key. | ✅ |
| `DB_HOST` | MySQL database host | ✅ |
| `DB_PORT` | MySQL database port | ✅ |
| `DB_NAME` | MySQL database name | ✅ |
| `DB_USER` | MySQL database username | ✅ |
| `DB_PASSWORD` | MySQL database password | ✅ |
| `FRONTEND_ORIGIN` | Allowed CORS origin | Default: `http://localhost:5173` |

**⚠️ Production checklist:**
- [ ] Set `JWT_SECRET` to a cryptographically random string ≥ 32 bytes.
- [ ] Set `ENCRYPTION_SECRET` to a unique, high-entropy string.
- [ ] Ensure `secrets.properties` is in `.gitignore` and never committed.
- [ ] Rotate `ENCRYPTION_SECRET` requires re-encrypting all existing `auth_tokens` rows.

## 6. Error Reference

### Global Auth Errors (from `AuthFilter`)

| Status | Error | Cause |
|--------|-------|-------|
| `401` | `"Missing authentication token"` | No `Authorization` header and no `?token=` query param |
| `401` | `"Invalid or expired token"` | JWT signature invalid, token expired, or malformed |

### OAuth Endpoint Errors

| Endpoint | Status | Error | Cause |
|----------|--------|-------|-------|
| `POST /oauth/link` | `400` | `"provider is required"` | Missing `provider` field |
| `POST /oauth/link` | `400` | `"client_id is required"` | Missing `client_id` field |
| `POST /oauth/link` | `400` | `"client_secret is required"` | Missing `client_secret` field |
| `POST /oauth/link` | `400` | `"token_endpoint is required"` | Missing `token_endpoint` field |
| `GET /oauth/callback` | `400` | `"Missing 'code' parameter"` | No authorization code in query |
| `GET /oauth/callback` | `400` | `"OAuth error from provider: ..."` | Provider returned an error (e.g., `access_denied`) |
| `GET /oauth/callback` | `400` | `"Cannot determine provider from state..."` | Malformed or missing `state` parameter |
| `POST /oauth/callback` | `400` | `"provider is required"` | Missing `provider` in request body |
| `POST /oauth/callback` | `400` | `"code (authorization code) is required"` | Missing `code` in request body |
| `POST /oauth/callback` | `400` | `"redirect_uri is required"` | Missing `redirect_uri` in request body |
| `PUT /oauth/link` | `400` | `"provider is required"` | Missing `provider` in request body |
| `DELETE /oauth/link` | `400` | `"provider query parameter is required"` | Missing `?provider=` query param |
